# History Mobile Chapter Separation

## Goal
Make chapter-intro blocks on the mobile History page read as "new chapter" markers, not as another regular information card.

## Problem
- Section intro blocks were too close in tone to the dark info cards below.
- On mobile, this reduced hierarchy: chapter start vs content block felt visually merged.

## Changes made
- Shifted mobile chapter-intro blocks to a lighter olive-stone treatment.
- Increased border presence and warmth so the block feels ceremonial and transitional.
- Added a subtle inner divider line near the bottom to suggest a chapter plaque rather than a standard content card.
- Increased spacing below the chapter-intro block so the following info card reads as a new layer.
- Slightly strengthened kicker and section-description styling for mobile readability.

## Visual intent
The page should now read like:
1. Navigation rail
2. Chapter plaque
3. Actual information card(s)

## Implementation
- `css/style.css`

## Notes
This change was targeted at the mobile History page only.
